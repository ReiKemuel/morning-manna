# Morning Manna — Process Audit (2026-07-15)

Double-check of the Day-1 pipeline (notes → edition JSON → renderer → Gmail draft), what was
fixed on the spot, and what to build next — each item mapped to AIS Challenge Days 2–7.

## Loopholes found

### 1. Personal name was in the PUBLIC repo (fixed, one decision left)
`tools/render_newsletter.py`'s docstring example carried the real preacher name + real sermon
meta — and it was committed and pushed. The "verified PII-free before each push" claim from
Day 1 missed it (manual eyeballing failed; 5th instance of the unverified-claim pattern).
- **Applied:** docstring now uses a placeholder; added `tools/check_repo_clean.sh` — greps
  tracked files for personal-name patterns + tracked content dirs, fails loud. Run it before
  every push (or wire it as `.git/hooks/pre-push`). The patterns themselves live in
  `.pii_patterns` (gitignored) so the guard never leaks what it guards.
- **Your decision:** the name is still in git *history* (commit `a6205d5`). Repo is 2 commits
  old with no forks — a history rewrite is cheap **today** and only gets more expensive.
  Proposed (needs your go-ahead, rewrites history + force-push):
  `git rebase -i --root` (or `git filter-repo`), then `git push --force`.
  Alternative: accept it — it's a pastor's name in a ministry context, arguably public.

### 2. Renderer rendered garbage silently (fixed)
Every field defaulted to `""` — a malformed edition produced a half-empty email with no error.
- **Applied:** `validate()` — required fields (title/date/service/text/intro/sections/
  memory_verse/application_prompt, plus per-section and verse sub-fields) fail loudly with a
  list of what's missing. Tested both paths. (IDEAS #13.)

### 3. Raw-HTML trust boundary — becomes real at Day 2 (not applied, by design)
`intro` and `sections[].body` pass through unescaped when they start with `<`. Safe while the
agent authors every string; the moment **Notion (Day 2)** or **YouTube (later)** feeds the
pipeline, external text can smuggle markup/links into your email. Fix belongs in the Day 2
build: sanitize at ingestion (allowlist `<p> <em> <strong>`) or escape-by-default.

### 4. Content durability — single point of failure (report only)
Editions are gitignored (correct, privacy-first), `.tmp/` is disposable, Gmail drafts get
trimmed. Net: the edition JSONs on this one Mac are the only durable copies, and Day 1's raw
notes were never retained, so faithfulness can't be re-verified. **Day 2 resolves this** —
Notion becomes the durable note source. Until then the exposure is small and acceptable.

### 5. Stale-draft risk in Gmail (your 2-minute action)
Drafts never expire and any of them can be sent with one tap. There are now 5 Morning Manna
drafts (3 Day-1 palette A/Bs + Monday + Wednesday). **Trim the 3 old A/Bs.** Convention going
forward: one live draft per edition, subject = `Morning Manna — Title (Day, date)` (both new
drafts follow it). This matters much more once Day 6 scheduled-send exists.

### 6. Voice-check is subjective and unlogged (Day 3 fixes)
Workflow step 5 ("does this sound like Rei?") has no checklist and no record. Day 3's Skill
should freeze the voice signatures (italics-for-inner-speech, Taglish warmth, one-thought
sentences, narrative connectors) into a testable checklist.

### 7. Small engine gaps (backlog, low stakes)
- Date is freeform text; nothing checks the JSON date matches the filename slug.
- `_p()` doesn't split multi-paragraph plain text (moot while bodies are authored as HTML).
- Dark-mode email variant still open (IDEAS #9). Size is fine (~11KB vs Gmail's 102KB clip).

## Recommendations mapped to Days 2–7

- **Day 2 — Notion MCP note source.** Doubles as the fix for #3 (sanitize at ingestion) and
  #4 (durable notes). Do first: define a note template in Notion (date, service, preacher,
  text, body) so extraction is mechanical, not guesswork.
- **Day 3 — Skill.** Freeze voice + edition schema + the voice-check checklist (#6), and
  encode the weekly-drip source table so "what feeds today's edition" is deterministic.
- **Day 4–6 prep — automate the continuity blocks.** `previous_application` and
  `recent_verses` are hand-filled today. Small Python step: auto-pull them from the previous
  edition JSON(s) in `editions/`. Tighter WAT; required before daily runs are hands-off.
- **Day 5 — collapsible web version + brand.** Also the natural slot for the **wordmark/logo**
  you asked to be reminded about (gold-on-cornflower, typographic).
- **Day 6 — scheduled send.** Design the approval loop explicitly: evening draft → your
  approve → scheduled AM send. The stale-draft convention (#5) becomes a hard requirement
  here — the sender must only ever see one approved draft.
- **Day 7 — EA integration.** Morning Manna as one routine; the continuity automation from
  Day 4–6 prep is what makes it a clean plug-in.
